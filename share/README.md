# 📋 請求書自動管理システム

## これは何？

毎月届く大量の請求書メール（200〜300件）を**全自動で処理**するシステムです。

Gmailに届いたPDF請求書を自動で：
1. 📧 **メールから検出**（PDF添付ファイルを自動抽出）
2. 📁 **Google Driveに保存**（年月別フォルダで整理）
3. 🤖 **AIで解析**（メーカー名＋税抜金額をGemini AIが読み取り）
4. 📋 **スプレッドシートに記入**（縦=メーカー名、横=月の一覧表）

```
処理の流れ:
Gmail → PDF抽出 → Drive保存 → AI解析 → スプレッドシート記入
```

---

## 🎯 できること

### スプレッドシート（自動作成）
メーカー名×月の一覧表が自動で作成されます：

|           | 2026/02  | 2026/03  | 2026/04  |
|-----------|----------|----------|----------|
| メーカーA  | 50,000   | 48,000   | 52,000   |
| メーカーB  | 120,000  | 120,000  | 115,000  |
| メーカーC  | 80,000   | 85,000   |          |

- 新しいメーカーは自動で行追加
- 新しい月は自動で列追加（時系列順にソート）
- 同じメーカーから複数請求書がある場合は自動加算

### Google Driveフォルダ（自動作成）
```
📁 請求書/
  📁 2026年02月/
    📄 メーカーA_請求書_202602.pdf
    📄 メーカーB_請求書_202602.pdf
  📁 2026年03月/
    📄 メーカーA_請求書_202603.pdf
    ...
```

### 安全機能
- **二重処理防止**: 処理済みメールに自動でGmailラベルを付与
- **処理ログ**: スプレッドシートの「処理ログ」シートに全記録
- **バッチ処理**: 1回30件ずつ処理（Google Apps Scriptの実行時間制限対策）

## ⏱ セットアップ手順（5分で完了）

### Step 1: Gemini API Key を取得（無料）
1. https://aistudio.google.com/apikey を開く（Googleアカウントでログイン）
2. 「APIキーを作成」をクリック
3. 表示されたキーをコピー（`AIza...`で始まる文字列）

> 💡 無料枠: 1日1500リクエストまで（月300件の請求書なら余裕）

### Step 2: Google Apps Script プロジェクト作成
1. https://script.google.com/ を開く
2. 「新しいプロジェクト」をクリック
3. 上部のプロジェクト名「無題のプロジェクト」をクリック → 「請求書管理」に変更

### Step 3: コードを貼り付け
1. 既存の `コード.gs` の中身を全選択（Ctrl+A）して、同梱の **コード.gs** の内容に丸ごと置き換え
2. 左サイドバーの「＋」→「スクリプト」をクリック
3. ファイル名を `設定` に変更
4. 同梱の **設定.gs** の内容を貼り付け
5. `設定.gs` の1行目 `GEMINI_API_KEY` に Step 1 で取得したキーを貼り付け

### Step 4: テスト実行
1. 上部のドロップダウンで `testOneEmail` を選択
2. ▶ 実行ボタンをクリック
3. 「このアプリは確認されていません」→「詳細」→「（安全ではないページ）に移動」→「許可」
4. 下部の「実行ログ」で結果を確認（✅が表示されれば成功）

### Step 5: 本番実行
1. ドロップダウンで `processInvoices` を選択 → ▶ 実行
2. 2026年2月以降のPDF添付メールが一括処理されます
3. 件数が多い場合は複数回実行（30件ずつ処理）

### Step 6: 月次自動実行の設定
1. ドロップダウンで `setupTrigger` を選択 → ▶ 実行
2. これで毎月1日の朝9時に自動実行されます

## 🔧 カスタマイズ（設定.gs を編集）

| 設定項目 | 説明 | 初期値 |
|---|---|---|
| `GEMINI_API_KEY` | Gemini API キー（必須） | 空 |
| `START_DATE` | 処理対象の開始日 | `2026/02/01` |
| `BATCH_SIZE` | 1回の実行で処理するメール数 | `30` |
| `GMAIL_QUERY` | Gmail検索クエリ | `has:attachment filename:pdf` |
| `PROCESSED_LABEL` | 処理済みラベル名 | `請求書処理済` |

### メール検索の絞り込み例
```javascript
// 特定の送信者だけに絞る場合
var GMAIL_QUERY = 'has:attachment filename:pdf from:invoices@example.com';

// 特定のラベルが付いたメールだけ処理する場合
var GMAIL_QUERY = 'has:attachment filename:pdf label:請求書';

// 件名に「請求」が含まれるメールだけ
var GMAIL_QUERY = 'has:attachment filename:pdf subject:請求';
```

## 🔄 便利な関数一覧

GASエディタの関数ドロップダウンから選んで ▶ 実行：

| 関数名 | 用途 |
|---|---|
| `processInvoices` | 未処理の請求書メールを一括処理（メイン関数） |
| `testOneEmail` | 1件だけテスト処理（初回動作確認用） |
| `setupTrigger` | 毎月1日の自動実行をON |
| `removeTrigger` | 自動実行をOFF |
| `resetProcessedLabel` | 全メールの処理済みラベルを解除（再処理したい時） |

## ❓ よくある質問

**Q: 請求書以外のPDFも処理される？**
A: PDF添付メール全てが対象ですが、AIが「請求書でない」と判断したものはスキップされます。より正確に絞りたい場合は `GMAIL_QUERY` をカスタマイズしてください。

**Q: 同じメーカーから月に複数の請求書が来る場合は？**
A: 自動で金額が加算されます。

**Q: 処理を間違えた場合は？**
A: スプレッドシートを直接編集してOKです。再処理する場合は `resetProcessedLabel` を実行後、`processInvoices` を再実行してください。

**Q: 無料枠を超えた場合は？**
A: Gemini APIの無料枠は1日1500リクエスト。超える場合は翌日に再実行するか、Google Cloud の課金を有効にしてください。

## 📊 処理件数とAPI制限について

### 「1件」とは？
**PDF 1ファイルの解析 = 1件**（＝1リクエスト）です。
- メール1通にPDF1つ → **1件**
- メール1通にPDF3つ → **3件**

### 無料枠の目安

| 項目 | 制限 |
|---|---|
| 1分あたり | 約10件 |
| 1日あたり | 約1,500件 |

> 💡 月200〜300件の請求書なら**無料枠で十分**です。

### 大量件数の場合

| 月の件数 | 対応方法 |
|---|---|
| 〜50件 | 1回の実行で完了 |
| 50〜200件 | 自動で30件ずつ分割処理。数回実行すればOK |
| 200件超 | 自動トリガー（毎月1日）で処理。1日で終わらなくても翌日自動で続きを処理 |
| 1000件超 | Google Cloudで課金を有効にすれば制限なし（月数十円程度） |

### 制限にかかった場合
- エラーになったメールは**スキップされるだけ**（データは壊れません）
- 処理済みメールにはラベルが付くので、次回実行時に**未処理分だけ**が対象
- 翌日に `processInvoices` を再実行するか、自動トリガーが毎月1日に再実行します
